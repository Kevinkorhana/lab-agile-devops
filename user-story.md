# User Story Template

## Title: [Short title describing the feature]

### User Story
**As a** [role / type of user],  
**I want** [the capability / action I can take],  
**So that** [the benefit, value, or reason for this feature].

---

### Acceptance Criteria
#### Scenario 1: [Successful Path / Happy Path]
* **Given** [the initial context or state of the system],
* **When** [the action or event occurs],
* **Then** [the expected outcome or response].

#### Scenario 2: [Alternative or Error Path]
* **Given** [the initial context],
* **When** [the invalid action or event occurs],
* **Then** [the system should handle it gracefully or show an error].
